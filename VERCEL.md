# Настройка Vercel для Olempic Panel

## Шаг 1: Аккаунт и репозиторий

1. Зайдите на **[vercel.com](https://vercel.com)** и войдите (через GitHub удобнее).
2. Убедитесь, что проект **olempic-panel** уже в GitHub (если нет — см. DEPLOY.md, раздел «Репозиторий на GitHub»).

---

## Шаг 2: Импорт проекта

1. В Vercel нажмите **Add New** → **Project**.
2. Выберите **Import Git Repository** и найдите репозиторий **olempic-panel**.
3. Если репозитория нет в списке — нажмите **Adjust GitHub App Permissions** и дайте Vercel доступ к нужному аккаунту/организации.

---

## Шаг 3: Настройки сборки

Перед деплоем проверьте:

| Поле | Значение |
|------|----------|
| **Framework Preset** | Other |
| **Root Directory** | оставьте пустым (или `.`) |
| **Build Command** | оставьте пустым |
| **Output Directory** | оставьте пустым |
| **Install Command** | оставьте пустым |

Нажмите **Deploy**.

---

## Шаг 4: После первого деплоя

- Сайт откроется по адресу вида **`olempic-panel-xxxx.vercel.app`**.
- Главная страница `/` будет показывать панель (благодаря `vercel.json`: корень перенаправляется на `dashboard.html`).

---

## Шаг 5: Автодеплой

После подключения репозитория каждый **push в ветку `main`** будет автоматически запускать новый деплой. Отдельно настраивать GitHub Actions для Vercel не нужно.

---

## Свой домен (по желанию)

1. В Vercel откройте проект → **Settings** → **Domains**.
2. Добавьте домен (например `panel.yourdomain.com`).
3. В настройках DNS у регистратора создайте запись по подсказкам Vercel (обычно CNAME на `cname.vercel-dns.com`).

---

## CORS (если панель зовёт API на 52.54.172.121)

Если фронт на Vercel, а API на другом домене, на бэкенде нужно разрешить запросы с домена Vercel:

- Добавить в CORS заголовки домен вида: `https://olempic-panel-xxxx.vercel.app`  
  или `https://*.vercel.app` (если бэкенд это поддерживает).

Иначе браузер будет блокировать запросы к API.
